BACKROOMS — WORLD MAP

A living map of every room built so far, and the documentation for the room-tracking system behind it. The diagram below is generated directly from the game's room data, so it is always in sync with what you can actually walk through.

THE MAP

Each box is a room, placed at its real position in the world grid: a room one step north sits directly above its neighbour, a room to the west sits to its left. The small nubs on a room's edges are doors — where two adjacent rooms both have a nub facing each other, you can walk between them. Hover a room for its exits; click to pin a highlight while tracing a path.

This page is the full design reference — every room built so far. The in-game map is fog-of-war: it reveals only the rooms the player has actually walked into, filling in as they explore, and resets to the spawn room whenever the game is refreshed or restarted.

Some rooms are scroll rooms: instead of a single screen, they stitch several screen-sized subrooms into one continuous space the camera scrolls across. Subrooms are joined by open walls — wide gaps you simply walk through, with a light, fast breeze rather than the creak of a door — and a scroll room can take on non-rectangular shapes as long as those openings line up. On both maps the whole thing is still a single cell: the subrooms (referred to internally as R7-1, R7-2, …) are an implementation detail the player never sees. You stay "in R7" and navigate it by scrolling.

In-game, a beacon flashes over the room the player is currently in — open it from Menu → World Map.

HOW IT WORKS

Rooms are defined once, in src/data/rooms.ts, as a list of SPECS — each with an id, a grid col/row, and which edges have doors. From that single list the game derives both the playable rooms and a lightweight ROOM_META array. The map UI (src/ui/roomMap.ts) renders purely from ROOM_META, using CSS grid to lay each room out at its col/row.

The same renderer powers two surfaces:

  • This page — a public reference map of the whole world.
  • The in-game dialog — opened from Menu → World Map, sized to the game screen, with a flashing beacon on the player's current room and an easy exit (× button, backdrop click, or Esc) back to the game. The beacon follows the player live via a small location store (src/game/systems/playerLocation.ts).

ADDING A ROOM

Because everything flows from the one SPECS list, adding a room puts it on this map and the in-game map automatically — there is no second place to update. Add an entry at the coordinate you want, wired to its neighbours with matching doors: 

// src/data/rooms.ts
{
  id: 'R7',
  col: 3,            // one room east of R4 (col 2)
  row: 1,
  doors: { west: true },   // a door back toward R4
},

For the connection to be walkable, the neighbour must have the opposing door too (here, give R4 an east: true door). Doors use standardized openings on both sides, so the player always lands in the matching gap when crossing. Reload the map and the new room appears in place.
